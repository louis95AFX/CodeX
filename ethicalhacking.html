Network Scanning with Nmap

Network scanning is a fundamental skill for ethical hackers. It allows you to discover devices, services, and vulnerabilities in a network. Nmap (Network Mapper) is the most popular tool for this purpose.

Legal Notice

Always obtain proper authorization before scanning any network. Unauthorized scanning is illegal and can result in severe penalties.

Basic Nmap Commands

Here are some essential Nmap commands every ethical hacker should know:

# Basic host discovery
nmap -sn 192.168.1.0/24
# Scan for open ports
nmap -sS 192.168.1.1
# Service version detection
nmap -sV 192.168.1.1
# OS detection
nmap -O 192.168.1.1

Understanding Scan Types

TCP SYN Scan (-sS)

The default and most popular scan option. It's fast, stealthy, and relatively unobtrusive since it doesn't complete TCP connections.

UDP Scan (-sU)

Important for discovering UDP services. Much slower than TCP scans because UDP is connectionless.

Comprehensive Scan (-A)

Enables OS detection, version detection, script scanning, and traceroute for a thorough examination.

Stealth Scan (-sS -T2)

Slower scan that avoids detection by intrusion detection systems.

Interpreting Results

Understanding Nmap output is crucial for effective security assessment:

Pro Tip

Combine Nmap with NSE (Nmap Scripting Engine) for advanced vulnerability detection. For example: nmap --script vuln 192.168.1.1
